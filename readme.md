Значения файлов:
* Path - классы описания пути и статические функции для манипуляций с ними
  * Путь - массив из Segment, в котором p1 и p2 - Point
  * Функции способны работать и с Segment, и с подклассами (SchedSegment)

* DOMBall, DOMPath, DOMBanner - управление svg-элементами на странице
  * DOMPath имеет подклассы SchedSegment и SchedPoint (со стилями и маской), рисует путь из них
  * Создаются без аргументов после загрузки страницы, пересоздавать не надо
  * Главная функция - Update. Без аргументов стирает/прячет элемент

* PathAnimation - проходит путь по кадрам

* NormalAnimator - берет траекторию из Segment и параметры маски, считает ее, 
преобразует путь в SchedSegment и запускает анимацию
  * Расчет маски реализован как разделение пути на 2 с разрезанием сегмента, пересекающего маску. 
  В итоге получается "лишняя" точка начала маски

* LoggedLaunchers - классы, запускающие конкретную парадигму. Принимают траекторию и объекты управления (DOM...),
запускаются через *async* Run
  * Ведут журнал (словарь) в journal

* RandomRunner - загружает траектории из библиотеки, перемешивает их в батчи и запускает парадигмы с ними

* TrajLibrary - библиотека траекторий

Типы заданий (парадигм):
1) Прямой путь без окклюдера, остановка пробелом
2) Прямой путь с маской на половину пути, остановка пробелом
3) Прямой путь без остановки, затем замер времени обратного
4) Воспроизведение интервала

__В тестах 3 и 4 есть ограничение - если не ответить в течение 10 секунд, попытка завершится сама__

Интервал можно исправить в LoggedLaunchers
```
await Promise.any([
    waitForSpace(), // wait for space
    new Promise(res => setTimeout(res, 10000, true)) // or 10s
]);
```
